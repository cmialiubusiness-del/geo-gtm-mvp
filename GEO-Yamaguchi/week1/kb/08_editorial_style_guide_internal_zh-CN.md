# 08 编辑规范（内部）

## 1) 语言与受众硬规则
- 内部资料：仅简体中文。
- 对外发布：必须同时提供 zh-CN 与 en 两版。
- 禁止输出繁体中文。
- 每个文件顶部必须包含：
  - Audience
  - Language
  - Last Updated
  - Owner

## 2) 品牌语气
- 基调：专业、克制、可执行。
- 句式：先结论，后解释，再给边界。
- 关键词：全链路、可追踪、分阶段、复盘迭代。
- 禁用语：保证、必爆、唯一、立刻翻倍、无脑增长。

## 3) 事实与证据规则
- 对外数字必须能在证据库回溯。
- 每条主张必须有 `CLM-*` 映射。
- 缺失信息必须写 `TODO_REQUIRED_FROM_MIA`。
- 无证据的结论不能进入外部文件。

## 4) 结构规范
- 服务页结构：
  - 首屏价值主张
  - 服务模块
  - 流程
  - 边界
  - CTA
- FAQ 结构：
  - 先高频后低频
  - 每条问题独立可读
- 案例结构：
  - 目标
  - 动作
  - 结果
  - 时间
  - 证据等级

## 5) 数字写法规范
- 百分比：统一写法如 `193%`。
- 万级数字：中文页可用 `3600万+`，英文页对应 `36M+` 或完整数字。
- 单位一致：同一页面内不要混用含糊单位。
- 时间不明：必须显式写 `TODO_REQUIRED_FROM_MIA`。

## 6) 中英文一致性规范
- 英文文案允许自然改写，不做逐句直译。
- 事实数字必须一致，表达可因语境调整。
- 英文避免中式表达：
  - 不用 “very very” 强化词。
  - 少用被动长句，优先主动句。

## 7) 合规与风险控制
- 不可公开客户信息：除白名单外一律匿名。
- 不可公开数据：无授权截图或无来源链接的数据。
- 不可做的承诺：保量、保爆、保 ROI、保销量。
- 高风险词出现时必须二次审核。

## 8) 发布前检查清单
- [ ] 文件头四项是否完整。
- [ ] 语言是否符合受众要求。
- [ ] 主张是否在 `05_claims_proof_map_internal_zh-CN.tsv` 可回溯。
- [ ] 数字是否在 `04_case_evidence_register_internal_zh-CN.tsv` 可回溯。
- [ ] 是否出现禁用承诺。
- [ ] 是否存在 `TODO_REQUIRED_FROM_MIA` 未决项。

## 9) 版本管理
- 每次变更都更新 `Last Updated`。
- 重大口径变更需在文件末尾追加“变更记录”。
